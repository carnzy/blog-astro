## 1. 前置知识

### 1.1 Astro 框架基础

本博客使用 **Astro** 框架构建。Astro 的核心概念：

- **`.astro` 文件**：包含组件模板（HTML-like）、前端脚本（`---` 隔离的 TypeScript）和客户端脚本（`<script>` 标签）
- **组件层级**：`Layout.astro` → `Navbar.astro` / `SideBar.astro` → 具体 Widget 组件
- **岛架构（Islands）**：静态 HTML + 按需加载的交互式组件

### 1.2 项目目录结构

```
src/
├── components/
│   ├── features/        # 功能组件（管理器、特效等）
│   ├── layout/          # 布局组件（导航栏、侧边栏等）
│   └── widget/          # 小部件组件（音乐、公告、分类等）
├── config/              # 配置文件
│   └── index.ts         # 统一导出
├── i18n/                # 国际化
│   ├── i18nKey.ts       # 翻译键定义
│   └── languages/       # 各语言翻译文件
├── layouts/             # 页面布局
│   └── Layout.astro     # 根布局
├── types/               # TypeScript 类型定义
│   └── config.ts        # 所有配置接口
├── global.d.ts          # 全局类型声明
└── env.d.ts             # 环境变量类型
```

### 1.3 音乐模块的代码路径

```
用户访问页面
    │
    ▼
Layout.astro  ← 加载 MusicManager（全局单例管理器）
    │
    ├── Navbar.astro  ← 导航栏音乐按钮 + 弹出面板（MusicPlayer）
    │
    └── SideBar.astro  ← 侧边栏音乐小部件（Music → MusicPlayer）
```

## 2. 架构分析：音乐模块如何工作

### 2.1 三层架构

音乐模块采用 **管理器-视图分离** 模式：

```
┌─────────────────────────────────┐
│     MusicManager.astro           │  ← 第1层：全局管理器
│  (Layout.astro 中加载，单例)      │
│                                   │
│  • 创建唯一的 <audio> 元素        │
│  • 管理播放状态、音量、播放模式    │
│  • 从 Meting API 或本地加载音乐    │
│  • 通过 window.__fireflyMusic     │
│    暴露公共 API                    │
│  • 通过 CustomEvent (fm:*) 广播   │
│    状态变更给所有视图实例           │
└──────────────┬──────────────────┘
               │ CustomEvent 通信
    ┌──────────┴──────────┐
    │                      │
    ▼                      ▼
┌──────────────┐   ┌──────────────┐
│ Music.astro   │   │ Music.astro   │  ← 第2层：小部件包装
│ (侧边栏)       │   │ (导航栏)       │
│              │   │              │
│ WidgetLayout │   │ 弹出面板      │
└──────┬───────┘   └──────┬───────┘
       │                   │
       ▼                   ▼
┌─────────────────────────────────┐
│     MusicPlayer.astro            │  ← 第3层：UI 视图
│                                   │
│  • 播放/暂停/上一首/下一首按钮      │
│  • 进度条（可拖拽）                │
│  • 音量控制                        │
│  • 歌词显示（LRC 解析和滚动同步）   │
│  • 播放列表                        │
│  • 封面图片旋转动画                │
│  • 通过 window.__fireflyMusic     │
│    调用管理器 API                  │
│  • 监听 fm:* CustomEvent 更新 UI  │
└─────────────────────────────────┘
```

### 2.2 数据流

```
配置 (musicConfig.ts)
    ↓ 导入
MusicManager.astro  (服务器端序列化配置 → JSON → 客户端解析)
    ↓
创建 __fireflyMusic 全局对象
    ↓
MusicPlayer.astro 挂载 → 读取 __fireflyMusic.getState() → 渲染 UI
    ↓
用户点击按钮 → __fireflyMusic.togglePlay() 等
    ↓
audio 元素事件 (timeupdate, ended, error)
    ↓
CustomEvent (fm:time, fm:play-state, fm:track, ...)
    ↓
MusicPlayer 监听事件 → 更新 DOM
```

### 2.3 关键依赖关系

```typescript
// 类型定义链
types/config.ts:
  WidgetComponentType = "profile" | "announcement" | ... | "music"
  MusicPlayerConfig = { mode, meting, local, ... }

// 配置链
config/musicConfig.ts → config/index.ts → 各组件导入

// 组件映射链  
SideBar.astro:
  componentMap = { profile: Profile, ..., music: Music }

// 全局类型链
global.d.ts:
  Window.__fireflyMusic = { init, getState, togglePlay, ... }
```

### 2.4 注册位置（音乐出现在哪些地方）

1. **侧边栏左侧** (`sidebarConfig.ts` → `leftComponents` → `{ type: "music" }`)
2. **导航栏** (`Navbar.astro` → `musicPlayerConfig.showInNavbar` 控制)
3. **移动端底部** (`sidebarConfig.ts` → `mobileBottomComponents` → `{ type: "music" }`)

## 关键原理总结

删除一个模块需要沿 **依赖链** 从底层到顶层逐步清理：

```
资源文件 → 配置文件 → 类型定义 → 视图组件 → 布局组件 → 全局声明 → 国际化
```

每个层次的清理逻辑：

| 层次 | 清理内容 | 原因 |
|------|---------|------|
| 资源文件 | 删除 mp3/封面等 | 避免冗余文件占用空间 |
| 配置文件 | 删除配置文件和导出 | 其他文件可能import该配置 |
| 类型定义 | 删除 interface/type | TypeScript 类型隔离 |
| 视图组件 | 删除 .astro 文件 | 核心功能载体 |
| 布局组件 | 删除 import 和实例化 | 页面中引用该组件 |
| 全局声明 | 删除 Window 接口扩展 | `window.__xxx` 类型安全 |
| 国际化 | 删除翻译键和值 | 所有语言文件需同步更新 |
| 文档 | 更新 README | 保持文档与代码一致 |

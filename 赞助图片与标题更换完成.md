## 如何手动完成这些修改（从 0 到 1 的技术原理）

### 修改横幅标题

1. **定位配置文件**: 找到 `src/config/backgroundWallpaper.ts`
2. **找到 `banner.homeText.title` 字段**: 这是 Astro 主题用于渲染首页横幅标题的配置项
3. **修改值**: 将 `"Lovely firefly!"` 改为 `"Fzy's Blog"`
4. **构建生效**: 运行 `npm run build` 或 `npm run dev`，Astro 会读取配置并在 `Layout.astro` 中渲染为 `.banner-title` 对应的 `<div>`

**原理**: 该主题使用集中配置模式（Convention over Configuration），所有 UI 文案和设置集中在 `src/config/` 目录下的 TS 文件中。`backgroundWallpaper.ts` 控制横幅壁纸、标题、轮播等所有 banner 相关行为。构建时 Astro/Vite 将这些配置内联到页面组件中。

### 更换赞助图片

1. **准备图片**: 将你的支付宝和微信收款码图片准备好
2. **放置图片**: 将图片放到 `public/assets/images/sponsor/` 目录下，命名为：
   - `alipay.png` — 支付宝收款码
   - `wechat.png` — 微信收款码
3. **确认配置**: 确保 `src/config/sponsorConfig.ts` 中 `qrCode` 字段指向正确路径（默认已是 `/assets/images/sponsor/alipay.png` 和 `/assets/images/sponsor/wechat.png`）
4. **构建生效**: 重新构建项目

**原理**: Astro 项目中 `public/` 目录下的文件会被直接复制到构建输出目录（`dist/`），不经过任何处理。`/assets/images/sponsor/alipay.png` 这个路径在构建后对应 `dist/assets/images/sponsor/alipay.png`。网页运行时，`<img src="/assets/images/sponsor/alipay.png">` 会直接从服务器加载该静态文件。

### 数据流总结

```
src/config/sponsorConfig.ts (配置源)
        ↓
    Astro 组件读取配置
        ↓
    渲染 <img src="/assets/images/sponsor/alipay.png">
        ↓
    浏览器请求 /assets/images/sponsor/alipay.png
        ↓
public/assets/images/sponsor/alipay.png (静态文件)
```
